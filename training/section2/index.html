Interactivity

The real fun stuff happens when users inteact with your page.

What does that mean?

When they click.

When they touch.

When they scroll.

When they move their mouse.

When they type.

When they speak.

When they move.

JavaScript can listen.

When an action happens, there can be a response.

These actions are called events.

JavaScript events.

  • click: The event occurs when the user clicks on an element
  • mouseover: The event occurs when the pointer is moved onto an element
  • keyup: The event occurs when the user releases a key
  • load: The event occurs when a document has been loaded
  • focus: The event occurs when an element gets focus

Variable basics.

Variables can hold different types of information, like words, numbers, and collections of data.

Variable names can not have spaces, start with a number, or be a reserved keyword.

At the time of these slides JavaScript has 43 reserved keywords.

(3 of 3)

Variable basics.

Different kinds of variables.


  // variable pattern - var (value can change, function scoped)
  var favorite_band = "Weezer";

  // variable pattern - let (value can change, block scoped)
  let favorite_band = "Weezer";

  // variable pattern - constant (value can not be changed)
  const favorite_band = "Weezer";

Really!?!

Const can't be changed?

How can I try that out?

To the console.

Something really important just happened!

We learned something new

We tested the idea without having to set up new environment or html page.

The console is GREAT for experimenting and learning.

What kind of variable should you use?

Current common convention.

If the value does NOT change, use const.

If the value does change, use let.

Avoid var if possible.

(2 of 3)

Logic basics.

Comparison Operators

Example Name Result
a == b Equal TRUE if a is equal to b (can be different types).
a === b Identical TRUE if a is equal to b, and the same type.
a != b Not equal TRUE if a is not equal to b (can be different types).
a !== b Not identical TRUE if a is not equal to b, or they are not the same type.
(3 of 3)

Logic basics.

Comparison Operators

Example Name Result
a < b Less than TRUE if a is strictly less than b.
a > b Greater than TRUE if a is strictly greater than b.
a <= b Less than or equal to TRUE if a is less than or equal to b.
a >= b Greater than or equal to TRUE if a is greater than or equal to b.

Using Functions

First, declare the function.

    }
    return result;
  }

  is_number_valid(6); // true
  is_number_valid(-2); // false
  is_number_valid("r"); // false
